Layout Overview

Website layouts should be designed to be comprehensible, coherent, and consistent. There are multiple WCAG standards that cover layout design. Here is a synopsis:


Information structure Level A

This is a heading.

<h4>This is a heading</h4>

An example of proper HTML tag usage.

Mark all layout elements with the proper HTML tags. For example, headings should always use <h> tags.

Tables should only be used for presenting tabular data and must be properly formatted.


Navigation and reading structureLevel A

  • Link 1
  • Link 2
  • Link 3

An example of breadcrumb navigation in the correct order.

The reading and navigation order of your content should be logical. For example, breadcrumb navigation should follow the correct sequence, and text content should be properly organized.


OrientationLevel AA

Responsive websites adapt to the user's screen shape.

The website layout must be responsive, meaning it is adaptable to any screen size or resolution.


Labels and IdentificationLevel AAA

Regions of the webpage should be properly identified using HTML labels or ARIA landmarks.
