Keyboard Overview

An accessible website can be navigated purely by a keyboard. The following guidelines will explain what needs to be done to ensure this.


Keyboard Functionality Level A

Every part of a page's functionaility is accessible using the keyboard. An exception to this is functionality that can't be accomplished with a keyboard (e.g. drawing)

Shortcut keys that are unique to your page or website should not interfere with existing browser and screen reader shortcuts.

Enhanced Keyboard FunctionalityLevel AAA

Every part of your page is available and functional using a keyboard.


Keyboard FocusLevel A

Keyboard focus should never become trapped or locked to a single page element. To test this, press tab on your keyboard while on the page. If the keyboard focus does not return to the first interactive element of the page, the test has failed.


Page-Defined ShortcutsLevel A

If you have designed a keyboard shortcut for a function that uses common characters (such as "P", "Q", ".", "[" and so on) then:

  • Include the functionality to disable the shortcut
  • Allow the user to change the shortcut to a non-printable key like Ctrl or Alt
  • Only activate the shortcut when it's associated element or button is focused
