Text Overview

Text is complex, because there are multiple elements to consider such as size, weight, line height, and more. Below you will find the guidelines you need to follow to create accessible type.


Presentation of TextLevel AAA

A graphic representing a block of text.

For blocks of text over one sentence in length:

  • No more than 80 characters wide
  • Are not fully justified (i.e. aligned to both margins)
  • Have line spacing of at least 1/2 the height of the text
  • Space between paragraphs is at least 1.5 times line spacing.

Specify a background and foreground color to text with CSS. This can be done individually to an element or page-wide.

Additionally, be sure that horizontal scrolling is not required to read the block of text when the text size is doubled, through magnification or other tools. In most cases horizontal scrolling can be prevented by properly setting up the "overflow" CSS trait of the text's parent element[s].


Text Spacing and ResizingLevel AA

A graphic representing a user resizing text.

Functionality or content is not lost when a user changes:

  • Paragraph spacing to 2 times the font size
  • Line height to 1.5 times the font size
  • Word spacing to 0.16 times the font size
  • Letter spacing to 0.12 times the font size

To best support the above guidelines, avoid using pixel height definitions for elements that contain text.

When resizing to 200%, the page must remain usable and readable.

Enhanced Text ResizingLevel AA

Content remains readable and functional when the website is resized to 320px in width. This would most likely require your design to be responsive.


Images of TextLevel AA

A graphic representing an image of text.

If a visiual representation is possible to be communicated by using text alone, then you should not use an image. For example, using an SVG graphic of a heavy-weighted font as a headline would break this standard.

Enhanced Images of TextLevel AAA

Text in an image is only purely decorative or essential to the information being shown.
